Warp メッセージは、必要最小限のデータのみを含むシンプルなフォーマットになっています。これは、効率的なデータ転送を可能にするためです。

- **NetworkID**

ネットワークの ID（Mainnet、Fuji テストネットワーク、ローカルテストネットワーク）。  
これは **セキュリティ対策** として含まれており、例えば Fuji テストネットワークのメッセージが Mainnet で再度実行されることを防ぎます。

- **SourceChainID**

メッセージの送信元となるチェーン ID。  
これは一般的な EVM のチェーン ID（ウォレットにネットワークを追加する際に使う ID）とは異なり、**P-Chain 上のブロックチェーン ID** です。  
P-Chain では、新しいブロックチェーンが作成された際のトランザクション ID がチェーン ID になります。  
例: `0xd7cdc6f08b167595d1577e24838113a88b1005b471a6c430d79c48b4c89cfc53`

- **SourceAddress** 

メッセージの送信元アドレス。  
どのアドレスがメッセージを発信したかを識別するために使用されます。

- **Payload**

メッセージ自体（データ）。  
送信したい情報や指示が含まれます。

---

各 Warp メッセージには **SHA256 ハッシュ** を用いた **一意の ID** が付与されます。  
この ID は、以下の要素をシリアライズ（データを一定の形式に変換）したものをハッシュ化することで生成されます。

> **Warp メッセージの ID = SHA256（NetworkID + SourceChainID + SourceAddress + Payload）**

---

このフィールドを含むメッセージは、まだ **署名されていない「UnsignedMessage」** の状態です。  
Warp メッセージの **セキュリティと認証を確保するためには、バリデーターによる署名が必要** です。
